# 1️⃣ 게임 수익성의 척도: ARPU & ARPPU


## ARPU (Average Revenue Per User)
**"유저 한 명이 평균적으로 얼마를 벌어다 주는가?"**

* **정의**: 전체 매출액을 **전체 유저 수(무과금 포함)**로 나눈 값
* **특징**: 게임의 전반적인 수익성을 가늠하는 지표
* **활용**: 캐주얼 게임처럼 유저 수는 많지만 결제액이 적은 경우 주로 활용

$$
ARPU = \frac{\text{Total Revenue (총 매출)}}{\text{Total Users (전체 유저 수)}}
$$

## ARPPU (Average Revenue Per Paying User)
**"결제한 유저들은 평균적으로 얼마를 쓰는가?"**

* **정의**: 전체 매출액을 **결제한 유저(Paying User) 수**로만 나눈 값
* **특징**: 실제 매출을 견인하는 핵심 유저층(고래 유저)의 구매력을 보여줌
* **활용**: MMORPG, 수집형 RPG 등 '헤비 유저'의 비중이 높은 게임에서 중요시함

$$
ARPPU = \frac{\text{Total Revenue (총 매출)}}{\text{Paying Users (결제 유저 수)}}
$$

### 📊 실전 예시로 비교하기

> **[상황 가정]**
> * **월 매출:** 5,000만 원
> * **월간 활성 유저(MAU):** 100,000명
> * **결제 유저(PU):** 1,000명

| 구분 | 계산식 | 결과값 | 해석 |
| :---: | :--- | :--- | :--- |
| **ARPU** | $50,000,000 \div 100,000$ | **500원** | 유저 1명당 500원의 가치를 가짐 |
| **ARPPU** | $50,000,000 \div 1,000$ | **50,000원** | 결제 유저는 평균 5만 원을 지출 |

> **💡**
> 위 예시처럼 ARPU와 ARPPU의 격차가 크다면(500원 vs 50,000원), 이 게임은 소수의 **고액 결제자(Heavy Spender)**에 의해 매출이 유지되고 있음을 알 수 있다.

# 노드 학습 풀이 - 1. ARPU, ARPPU 구하기

유저들의 구매 정보를 기록한 `arpu` 테이블을 활용해 ARPU, ARPPU를 각각 계산하라.

<details>
<summary>Query (클릭하면 열립니다)</summary>
<div markdown="1">

```sql
SELECT SUM(revenue) / COUNT(DISTINCT user_id) AS ARPU,
       SUM(revenue) / COUNT(DISTINCT CASE WHEN revenue > 0 THEN user_id END) AS ARPPU
FROM continual-nomad-479202-p8.modulabs.arpu
```

- `COUNT(DISTINCT user_id)`: 기간 내 접속한 전체 유저(Active Users)의 규모를 파악하기 위해 중복을 제거하고 카운트 (ARPU의 분모)

- `CASE WHEN revenue > 0 ...`: 전체 유저 중 **실제 매출이 발생한 유저(Paying Users)**만 필터링하는 조건부 집계 구문입니다. (ARPPU의 분모)

- `SUM(revenue)`: 해당 기간 동안 발생한 **총 매출액(Total Revenue)**입니다.

💡 Tip: `COUNT` 함수는 `NULL` 값을 세지 않는 특성이 있습니다. 따라서 `CASE WHEN` 구문에서 `ELSE`를 생략하면 조건에 맞지 않는 유저는 `NULL`이 되어 자동으로 카운트에서 제외됩니다. 이를 이용해 깔끔하게 결제 유저만 발라낼 수 있습니다.

</div>
</details>

# 2️⃣ 유저의 여정을 추적하다: 퍼널 분석 (Funnel Analysis)



![Image of marketing funnel analysis diagram](/assets/img/AIDA.jpg)_출처:https://blog.effic.biz/aidamodel_


퍼널 분석은 유저가 게임에 처음 들어와서(유입), 최종 목표(결제 등)에 도달하기까지의 과정을 **깔때기(Funnel)** 모양으로 시각화하여 분석하는 기법이다.

어떤 단계에서 유저가 가장 많이 이탈하는지 파악하여 **이탈률(Churn Rate)**을 줄이고 **전환율(Conversion Rate)**을 높이는 것이 목적이다. 

## AIDA 모델
고객의 **심리적 변화**에 초점을 맞춘 고전적인 마케팅 모델입니다.

| 단계 | 의미 | 게임 내 실제 사례 |
| :--- | :--- | :--- |
| **A**ttention (주의) | "이 게임 뭐지?" | 유튜브 광고, 앱스토어 피처드, 인플루언서 리뷰 노출 |
| **I**nterest (흥미) | "재밌어 보이네?" | 티저 영상 시청, 게임 스크린샷 확인, 상세 페이지 조회 |
| **D**esire (욕구) | "나도 하고 싶다!" | 사전 예약 보상 확인, 한정판 스킨 이벤트 등을 보고 설치 욕구 발생 |
| **A**ction (행동) | "설치 완료!" | **게임 다운로드, 설치, 회원가입 완료** |

<br>

## AARRR 모델 
스타트업과 앱 서비스에서 사용하는 **행동 데이터 중심**의 분석 프레임워크이다. 게임 서비스의 실질적인 성장을 측정하는 데 최적화되어 있다.



![Image of AARRR funnel model diagram](/assets/img/AARRR.jpg)
_출처:https://blog.dighty.com/business/?bmode=view&idx=11164135_

### 1. Acquisition (획득)
* **질문:** 유저가 우리 게임을 어떻게 알고 들어왔는가?
* **지표:** 앱스토어 검색 유입, 광고 채널별 다운로드 수(CPI)

### 2. Activation (활성화)
* **질문:** 유저가 게임의 핵심 재미를 느꼈는가?
* **지표:** 튜토리얼 완료율, 첫 번째 던전 클리어, 레벨 5 달성

### 3. Retention (유지)
* **질문:** 유저가 게임에 지속적으로 방문하는가?
* **지표:** D+1, D+7, D+30 리텐션(잔존율), 재접속률

### 4. Revenue (수익)
* **질문:** 유저가 돈을 쓰는가?
* **지표:** 인앱 결제(IAP), 광고 시청 매출, 구독권 구매 (LTV, ARPU)

### 5. Referral (추천)
* **질문:** 유저가 주변에 게임을 알리는가?
* **지표:** 친구 초대 횟수, SNS 공유 수, 길드/클랜 초대

# 3️⃣ AIDA vs AARRR 

두 모델은 비슷해 보이지만, 분석하고자 하는 **초점(Focus)**이 다르다.

| 모델 | AIDA  | AARRR  |
| :---: | :--- | :--- |
| **핵심** | **심리 변화** (마음) | **행동 데이터** (로그) |
| **활용** | 마케팅 메시지, 광고 소재, UI 기획 | 서비스 운영, 리텐션 개선, 수익화 전략 |
| **목적** | "어떻게 유입시킬 것인가?" | "어떻게 남기고 벌 것인가?" |

# 노드 학습 풀이 - 2. 퍼널 분석 : 사람들은 어디에서 이탈하고 있을까?

유저들의 액션 로그 정보를 기록한 `funnel` 테이블을 활용하여  퍼널 분석을 해봅시다.

각 페이지 단위의 방문 수와 `visits` 수를 모수로 한 페이지 단위의 전환율을 계산해 주세요.

<details>
<summary>Query (클릭하면 열립니다)</summary>
<div markdown="1">

```sql
#[[YOUR QUERY]]
WITH Funnel_Counts AS (
  SELECT
    COUNT(DISTINCT CASE WHEN action = 'visit' THEN user_id END) AS Visits,
    COUNT(DISTINCT CASE WHEN action = 'signup' THEN user_id END) AS Signups,
    COUNT(DISTINCT CASE WHEN action = 'add_to_cart' THEN user_id END) AS AddToCarts,
    COUNT(DISTINCT CASE WHEN action = 'purchase' THEN user_id END) AS Purchases
  FROM `continual-nomad-479202-p8.modulabs.funnel`
)

SELECT
  Visits,
  Signups,
  AddToCarts,
  Purchases,

  ROUND(Signups / Visits * 100, 2) AS SignupRate,          
  ROUND(AddToCarts / Visits * 100, 2) AS AddToCartRate,    
  ROUND(Purchases / Visits * 100, 2) AS PurchaseRate      
FROM Funnel_Counts;
```
- CTE(Common Table Expression) 활용: `WITH` 문으로 임시 테이블을 정의하여 가독성을 높이고, 메인 쿼리를 단순화함.

- 데이터 피벗(Pivot) 및 집계: `COUNT(DISTINCT CASE WHEN ...)` 구문을 활용하여, Row(행) 형태의 로그 데이터를 Column(열) 형태의 지표로 변환하고 중복 유저를 제거함.

- 전체 전환율 산출: 메인 쿼리에서 가장 상위 단계인 `Visits`를 분모로 설정하여, **유입 대비 각 단계별 도달률(Overall Conversion Rate)**을 백분율로 계산함.

</div>
</details>
