## 1️⃣ On-Boarding: 협업을 위한 도구 세팅

모두의연구소 과정은 **'퍼실리테이터(Facilitator)'** 님들의 가이드로 시작되었다. 단순한 강사가 아닌, 훈련생들의 러닝메이트로서 적응을 도와주시는 점이 인상 깊었다.

특히 이번 과정은 **100% 비대면**으로 진행되는 만큼, 다양한 협업 및 커뮤니케이션 툴을 익히는 것이 무엇보다 중요했다.

### 🛠️ 우리가 사용하는 툴(Tool) 정리

| 플랫폼 | 용도 | 주요 활동 |
| :--- | :--- | :--- |
| **ZEP** | 메타버스 강의장 | 출석 체크, 조회/종례, 아이스브레이킹 및 실시간 소통 |
| **Notion** | 메인 게시판 | 공지사항 확인, 학습 내용 정리(Wiki), 팀 협업 과제 수행 |
| **Discord** | 커뮤니티 | 실시간 질의응답, **[AI 뉴스]** 세션 (매주 화/목 최신 트렌드 공유) |
| **LMS** | 학습 관리 시스템 | 동영상 강의 시청, 클라우드 실습 환경 제공 |

> 
> 단순히 툴 사용법을 익히는 것을 넘어, **"비대면 환경에서도 어떻게 효율적으로 소통하고 협업할 것인가"**에 대한 감을 잡을 수 있었다.

## 2️⃣ 커리어 세션: 데이터 사이언티스트의 전망

본격적인 수업에 앞서, 현직 데이터 사이언티스트를 연사로 모시고 **Zoom 특강**을 진행했다.

* **주요 내용:** IT/AI/Data 직무 트렌드, 직무별(Data Analyst vs Scientist vs Engineer) 역할 차이
* **느낀 점:** 막연하게 알고 있던 직무의 경계를 명확히 할 수 있었고, 현재 산업 동향에 맞춰 어떤 역량을 길러야 할지 로드맵을 그려보는 시간이었다. 평소 궁금했던 점들을 현직자에게 직접 질문하며 해소할 수 있어 유익했다.

## 3️⃣ Tech Stack: 기초 체력 다지기

이번 주차의 핵심 학습 목표는 **"개발 환경에 익숙해지는 것"**이었다.

### 🐧 Linux (Terminal 환경)
마우스가 아닌 키보드 명령어로 파일을 관리하는 터미널 환경을 실습했다.
* **학습 내용:** `ls`, `cd`, `mkdir`, `rm` 등 기본 명령어 실습

학부 시절 전공 수업에서 OS 환경 자체가 리눅스였기 때문에 매우 익숙했다. 덕분에 막힘없이 실습을 진행하며 동료들을 도와줄 여유도 있었다. CLI(Command Line Interface) 환경의 효율성을 다시 한번 체감했다.

### 🐱 Github (버전 관리)
협업의 필수품인 Git과 Github의 워크플로우를 익혔다.

```bash
# 주요 실습 명령어 흐름
git init          # 저장소 초기화
git add .         # 변경사항 스테이징
git commit -m "msg" # 버전 기록
git push origin main # 원격 저장소 업로드
```

예전에 사용해 본 경험은 있지만, 관리가 안 되어 방치된 상태였다. 먼지 쌓인 나의 깃허브 페이지를 이번 기회에 싹 정리하고, 1일 1커밋을 위한 기반을 마련했다. Pull Request와 Issue 관리 등 협업 기능도 다시 복기했다.
