Python을 기초로 시작해 심화적인 내용까지 쭉 배워가며 실습도 진행했다.

# 3주차 (11/24 ~ 11/28) : SQL
<br>

Python을 마치고나서는 데이터베이스를 기술하는 SQL을 배우게 되었다.

사실 SQL도 조금 공부해 본 적이 있지만, 정말 얕게만 배워서 쌩기초 밖에 잘 모른다. 그래서 학습 초반에는 기억을 되살려가며 기본 문법 구조를 다지고, 새로운 내용을 공부하였다.

![sql_book](/assets/img/sql_book.png)
_정말 오랜만에 꺼내보는 책..._

그리고 훈련과정에서 Main Quest로 분류되는 첫 프로젝트? 느낌으로 [고객 세그먼테이션](https://chankyu99.github.io/posts/Main_Quest_SQL/) 프로젝트를 진행하였다.

SQL을 배우면서 가장 어려웠던 것은 **Window function**이었다. 

실무에서 많이 쓰이는 함수였기에, 더더욱 중요한 개념인 Window function은 기본적인 문법 구조는 단순하다. 

그러나 내가 막상 예시 데이터 테이블로 연습을 하려면 어떻게 해야할 지 감이 잡히지 않았다. 그래서 Python 클래스를 공부했던 것과 마찬가지로 이부분을 좀 더 깊이 파보기로 하였다. 

예시 테이블로 쿼리 구문을 직접 여러개 짜보면서 기능을 익히고, 실제 출력 결과와 예상 출력 결과를 직접 비교해보며 Window function이 어떤 기능을 하고 의미가 무엇인지 생각해 보았다. 아직도 조금 헷갈리지만, 열심히 하다보면 언젠가 자유자재로 쓰게 될 것 같다.

Window function을 정리한 [글](https://chankyu99.github.io/posts/WINDOWFUNCTION/)은 이곳에 정리하였다.

# 4주차 (12/01 ~ 12/05) : 비정제 데이터 다루기
<br>
2025년의 마지막 달과 함께 첫 번째 단위기간의 마지막 주가 되었다.

4주차에는 여러가지 데이터셋을 다루며 데이터를 전처리 하고, 스케일링과 엔지니어링을 통해 데이터를 정제하는 과정을 배웠다.

![data_cleaning](/assets/img/data_cleaning.png)
_데이터는 항상 깔끔하게 정제되어 있지 않다._

데이터셋이 항상 데이터 사이언티스트가 원하는 대로 수집이 되어 깔끔하게 정제되어 있다면 어떨까? 업무의 효율성이 말도 안되게 높아질 것이다. 

그러나 현실은 이상과 다르다. 모든 데이터셋을 우리가 원하는 대로 정제하려면 그것을 위한 전처리 과정이 필요하다. 

데이터를 정제하는 방법을 배우며 Main Quest 2번째로 [신용거래 이상탐지 데이터 다루기](http://chankyu99.github.io/posts/Main_Quest2_data_preprocessing/)를 진행하였다. 

프로젝트를 통해 어떻게 하면 예측 모델링에 필요한 성능향상을 기여할 수 있는 특성을 엔지니어링 할 수 있을지, 어떤 특성을 조합해야 가장 좋을지 고민하며 최적의 인사이트를 찾는 분석력을 기를 수 있었다.

비정제 데이터를 다루는 것을 주제로 강의해주신분이 꽤 낯설게 느껴졌는데, 알고 보니 예전에 데이터 사이언티스트 직무에 관심이 생겨 유튜브에서 정보를 찾다가 알게된 것이 기억이 났다. 

![youtube](/assets/img/youtube.png)
_출처 : 유튜브 [데싸노트](https://youtu.be/6XP9tIYph_Q?si=r2fxMdQs_jnBxLho)_

현직자로부터 데이터를 정제하는 방법을 배우고, 실무에서 유용한 기술들을 배울 수 있어 매우 유용한 노드학습 과정이라고 생각했다. 

다만, 굉장히 자세히 알려주시다보니 학습내용이 방대해서 4주차가 가장 따라가기 벅찬 과정이라고 생각했다. 

그래서 내가 놓쳤다고 생각할 수 있는 부분과 실무에서 유용한 함수를 정리한 [글](https://chankyu99.github.io/posts/Data_Cleaning/)을 포스트할 예정이다. 

이걸로 부족한 부분을 빨리 메꾸어야 다음에 배울 과정에 무리없이 따라갈 수 있을 것이다.
